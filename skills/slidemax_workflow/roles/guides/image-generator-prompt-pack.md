# Image Generator Prompt-Pack Guide

Use this guide when creating `images/image_prompts.md` and localizing image assets.

## Output Contract

The Image_Generator stage should leave the project with:

- `images/image_prompts.md` written before live generation starts
- Asset source type recorded for every required image
- Local project paths for generated, downloaded, or registered assets
- Enough metadata for Executor to reference images without guessing

## Asset Source Types

| Source Type | Meaning | Canonical Path |
|-------------|---------|----------------|
| `ai` | Image will be generated by a provider | `images/` |
| `local` | User or team already has the file | `images/` |
| `stock` | Commercial stock asset with provenance | `images/stock/` |
| `placeholder` | No file will be created yet | No local file required |

## Prompt-Pack File Structure

The prompt pack should contain:

1. Project header
2. Asset overview table
3. One section per asset
4. Usage notes

Per-asset section fields:

- Intended usage
- Visual type
- Source type
- Target local path
- Aspect ratio or image size
- Original description
- Prompt
- Negative prompt
- Alt text
- Acquisition notes

## Image Type Guidance

| Type | Typical Prompt Intent |
|------|------------------------|
| Background | Leave negative space for text overlay, avoid a dominant subject |
| Photography | Emphasize realism, lighting, and camera framing |
| Illustration | Emphasize style, simplified shapes, and palette control |
| Diagram | Emphasize structure, legibility, and low clutter |
| Decorative pattern | Emphasize subtlety and repeat-safe composition |

## Live Acquisition Rules

- Write the prompt pack before running any live generation.
- Smoke test the provider before trusting it for delivery work.
- Keep generated assets inside the project.
- Keep stock provenance inside `images/stock/manifest.json`.
- Do not leave remote links in delivery-ready slide assets.

## Canonical Commands

```bash
python3 skills/slidemax_workflow/scripts/slidemax.py image_generate "prompt" --provider gemini --aspect-ratio 16:9 --image-size 4K --output <project-path>/images --filename cover_bg
python3 skills/slidemax_workflow/scripts/slidemax.py smoke_test_image_provider --provider gemini --output <project-path>/images
python3 skills/slidemax_workflow/scripts/slidemax.py download_stock_image <project-path> --provider <provider> --source-url <url> --download-url <url> --filename <filename>
python3 skills/slidemax_workflow/scripts/slidemax.py register_stock_image <project-path> --provider <provider> --source-url <url> --local-file <path> --filename <filename>
```

## Negative Prompt Baseline

Use a type-specific negative prompt when needed. Default baseline:

```text
text, watermark, signature, blurry, distorted, low quality
```

## Completion Checklist

- [ ] `images/image_prompts.md` exists
- [ ] Every asset has source type, target path, and acquisition notes
- [ ] AI assets have prompt, negative prompt, and alt text
- [ ] Stock assets have provenance instructions or a registered manifest entry
- [ ] The next role can reference project-local paths directly
